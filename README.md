# stock_educational_project

Проект реализует базу данных для складского учёта на PostgreSQL: закупки от поставщиков, отгрузки со складов, контроль остатков, журналирование движений товаров и аналитические представления.

## Что делает проект

Система позволяет:

- хранить справочники товаров, категорий, поставщиков и складов;
- оформлять закупочные заказы (`purchase_orders`) и позиции заказов (`purchase_order_items`);
- оформлять отгрузки (`shipments`) и позиции отгрузок (`shipment_items`);
- автоматически пересчитывать остатки на складе;
- логировать входящие и исходящие движения товаров в `inventory_movements`;
- контролировать допустимые переходы статусов закупочных заказов;
- запрещать отгрузку товара, если на дату отгрузки недостаточно остатков;
- получать остатки на конкретную дату;
- строить отчёты через обычные и материализованные представления.

## Бизнес-логика

### Закупочные заказы

Для заказов поставщикам используется последовательность статусов:

```text
created → approved → formed → in transit → ready for issuance → completed
```

Файл `validate_po_status_transition.sql` создаёт триггер, который запрещает недопустимые переходы статусов.

Файл `allow_insert_items_only_for_created_po.sql` запрещает добавлять товары в заказ, если его статус уже отличается от `created`.

Когда заказ переходит в статус `completed`, триггер из `update_stock_on_po_completed.sql` добавляет товары из заказа в таблицу `stock`.

### Отгрузки

При переводе отгрузки в статус `shipped` триггер из `update_stock_on_shipment_shipped.sql` уменьшает остатки на складе.

Если отгрузка была в статусе `shipped`, а затем стала `canceled`, триггер из `update_stock_on_sts_canceled.sql` возвращает товары обратно на склад.

Перед добавлением или изменением позиции отгрузки триггер из `check_stock_quantity.sql` проверяет, что на складе достаточно товара на дату отгрузки.

### Журнал движений

Проект ведёт журнал движений товаров в таблице `inventory_movements`.

- `log_po_movements.sql` добавляет входящее движение (`type = 'in'`) при добавлении позиции закупочного заказа.
- `log_shipment_movement.sql` добавляет исходящее движение (`type = 'out'`) при добавлении позиции отгрузки.

## Представления и отчёты

Файл `view_stock.sql` создаёт:

| Представление | Описание |
|---|---|
| `stock_view` | Текущие остатки с названиями товаров и складов |
| `orders_from_suppliers` | Заказы поставщикам с товарами и последним статусом |
| `shipments_view` | Отгрузки со склада с товарами и статусами |
| `p_low_stock` | Товары, остаток которых меньше или равен минимальному запасу |
| `mv_incoming_by_day` | Приход товаров по дням |
| `mv_outcoming_by_day` | Расход товаров по дням |
| `mv_product_turnover` | Оборот товаров: приход, расход, баланс |
| `mv_supplier_activity` | Активность поставщиков по количеству заказов и товаров |

Для обновления материализованных представлений используйте:

```sql
REFRESH MATERIALIZED VIEW mv_incoming_by_day;
REFRESH MATERIALIZED VIEW mv_outcoming_by_day;
REFRESH MATERIALIZED VIEW mv_product_turnover;
REFRESH MATERIALIZED VIEW mv_supplier_activity;
```

## Функция получения остатков на дату

Функция `get_stock_on_date(p_date timestamptz)` возвращает остатки товаров на складах на выбранную дату.

Пример:

```sql
SELECT *
FROM get_stock_on_date('2027-02-01 00:00:00+00');
```

Результат содержит:

| Поле | Описание |
|---|---|
| `warehouse_id` | ID склада |
| `product_id` | ID товара |
| `qty_in` | Количество поступившего товара |
| `qty_out` | Количество отгруженного товара |
| `balance` | Остаток на дату |


## Примеры запросов

Посмотреть текущие остатки:

```sql
SELECT *
FROM stock_view;
```

Найти товары с низким остатком:

```sql
SELECT *
FROM p_low_stock;
```

Посмотреть отгрузки:

```sql
SELECT *
FROM shipments_view;
```

Посмотреть заказы поставщикам:

```sql
SELECT *
FROM orders_from_suppliers;
```

Получить оборот товаров:

```sql
SELECT *
FROM mv_product_turnover;
```

Проверить активность поставщиков:

```sql
SELECT *
FROM mv_supplier_activity;
```

## Индексы

Файл `indexes.sql` создаёт индексы для ускорения:

- поиска товара по `product_id`;
- поиска остатков по складу и товару;
- фильтрации движений по дате;
- фильтрации заказов и отгрузок по статусам;
- выборки завершённых заказов по дате;
- выборки отправленных отгрузок по дате.
